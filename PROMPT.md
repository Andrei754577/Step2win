# Промт для генерации сайта Step2Win (для другого ИИ)

Скопируй и отправь целиком следующий текст:

---

Создай полноценное веб-приложение (frontend + backend + база данных) — образовательную
платформу «Step2Win» для школьного IT Hub в Казахстане. Нужен рабочий MVP без ошибок,
с простым и понятным дизайном, масштабируемой архитектурой (должен выдерживать 1000+
одновременных запросов без зависаний).

## Технологический стек
- Backend: TypeScript (Node.js), REST API
- Frontend: React 18+ (Vite), TypeScript
- База данных: PostgreSQL или SQLite (с ORM — Drizzle или Prisma)
- Аутентификация: email + пароль (сессии/JWT), без соцсетей и без eGov/MFA в MVP
- Стилизация: Tailwind CSS
- Роутинг: любой SPA-роутер (React Router / Wouter)
- Управление состоянием запросов: TanStack Query (React Query)
- Мобильное приложение (отдельно, если возможно): Expo / React Native на том же REST API

## Языки интерфейса
Только русский и казахский (переключатель RU/KZ в шапке сайта), без английского.

## Роли пользователей
Только две роли: `student` (ученик) и `admin` (администратор). Отдельной роли
«наставник» нет — наставник указывается как обычное текстовое поле (имя, контакт)
внутри проекта.

## Дизайн
- Простой, чистый, интуитивный UI, mobile-first
- Основной цвет — синий (#2563EB), акцент — янтарный (#F59E0B), успех — зелёный,
  ошибка — красный, фон светло-серо-голубой, карточки белые со скруглением ~12px
- Шрифты: заголовки — Poppins (полужирный), текст — Inter
- Минимум décor, много воздуха, понятная иерархия

## Структура данных (основные таблицы)
- **users** — id, name, email, passwordHash, role(student/admin), xp, grade(класс 5-11),
  direction(направление), language
- **courses** — id, titleRu, titleKk, descriptionRu, descriptionKk, direction, coverColor,
  published
- **lessons** — id, courseId, titleRu, titleKk, contentRu, contentKk, order, xpReward
- **enrollments** — id, userId, courseId, progress(0-100%), status(active/completed)
- **lessonProgress** — id, userId, lessonId, completed, completedAt
- **certificates** — id, userId, courseId, code, issuedAt (выдаётся автоматически при 100%
  прохождении курса)
- **tests** — id, courseId, titleRu, titleKk, passScore(% для прохождения)
- **questions** — id, testId, textRu, textKk, type(single/multi), options(json массив),
  correctAnswer(json массив), order
- **testAttempts** — id, userId, testId, score, passed, answers(json), createdAt
- **projects** — id, title, description, stage(idea/hypothesis/mvp/demoday), direction,
  ownerId, mentorName, mentorContact, createdAt
- **projectUpdates** — id, projectId, authorId, message, createdAt
- **events** — id, titleRu, titleKk, descriptionRu, descriptionKk, type(hackathon/forum/olympiad),
  date, location, capacity
- **eventRegistrations** — id, eventId, userId, createdAt

## Функциональность (модули)

### 1. Лендинг (публичная главная страница)
Контент о проекте Step2Win — образовательный IT Hub, объединяющий школьников города
для развития цифровых навыков, проектного мышления и предпринимательских компетенций.
Связующее звено между школами, IT-экосистемами и площадками Astana Hub и
«Терриконовая долина» (Караганда, Казахстан).
Секции: hero с призывом к действию, статистика (500+ учащихся, 120+ проектов, 20+
форматов, 8 направлений), миссия, «о проекте», 8 направлений развития
(программирование, web-разработка, мобильная разработка, робототехника, ИИ,
кибербезопасность, дизайн, стартапы), этапы реализации проекта (4 этапа), блок
результатов (социальные/образовательные/экономические), финальный CTA.

### 2. Регистрация / авторизация
Email + пароль. При регистрации: имя, email, пароль, класс (5-11), направление.
Роль по умолчанию — student.

### 3. LMS (курсы)
- Список курсов с фильтром по направлению
- Страница курса: описание, список уроков, кнопка «Записаться»
- Прогресс по курсу (% пройденных уроков), XP за каждый урок
- При 100% прохождении курса — автоматическая выдача сертификата с уникальным кодом
- Личный кабинет: мои курсы, мой XP, мои сертификаты

### 4. Тестирование
- Банк вопросов с одним или несколькими правильными ответами
- Автоматическая проверка и подсчёт результата (% правильных ответов)
- Порог прохождения теста (passScore), результат «пройден/не пройден»

### 5. Проектный офис
- Доска проектов по этапам: идея → гипотеза → MVP → демодень
- Создание проекта учеником (название, описание)
- Поле «наставник» (имя/контакт, редактируется владельцем/админом)
- Лог обновлений/комментариев по проекту
- Смена этапа проекта владельцем или админом

### 6. Мероприятия
- Список мероприятий (хакатоны, форумы, олимпиады) с датой, местом, вместимостью
- Регистрация пользователя на мероприятие в один клик

### 7. Админ-панель (только для role=admin)
- Управление курсами и уроками (создание/удаление)
- Управление тестами и вопросами
- Управление мероприятиями
- Управление пользователями: список, смена роли (student/admin)

## Нефункциональные требования
- Время отклика интерфейса ≤ 2 сек
- Архитектура должна выдерживать не менее 1000 одновременных пользователей без
  деградации (эффективный event-loop backend, connection pooling к БД, без
  блокирующих синхронных операций в запросах)
- Все API-запросы возвращают понятные ошибки и статус-коды, есть базовая валидация
  входных данных
- Защищённые роуты (личный кабинет, админка) требуют авторизации; админка — только
  для role=admin
- Адаптивная вёрстка 320px–2560px

## Результат
Выдай полностью рабочий код (backend + frontend), готовый к локальному запуску одной
командой, с инструкцией по установке, накатке схемы БД и созданию тестового
администратора.
