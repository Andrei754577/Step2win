# Step2Win — образовательный IT Hub

MVP-версия платформы: веб-сайт + мобильное приложение (Expo/React Native) на едином бэкенде.

## Что реализовано

- **Лендинг** — контент о проекте (миссия, направления, этапы, показатели) на русском и казахском
- **Регистрация/авторизация** — email + пароль (Better Auth), роли: `student`, `admin`
- **LMS** — курсы, уроки, прогресс по курсу, начисление XP за уроки, автоматическая выдача сертификата при 100% прохождении курса
- **Тестирование** — банк вопросов (один/несколько правильных ответов), автопроверка, порог прохождения
- **Проектный офис** — доска идея → гипотеза → MVP → демодень, поле наставника (имя/контакт), лог обновлений по проекту
- **Мероприятия** — список хакатонов/форумов/олимпиад, регистрация
- **Админ-панель** — управление курсами и уроками, тестами, мероприятиями, ролями пользователей
- **RU/KK** — переключатель языка интерфейса на вебе и в мобильном приложении

### Сознательно не входит в MVP (по ТЗ, отложено)
- Авторизация через eGov Mobile / QR / MFA — сейчас email+пароль
- Роль "Наставник" как отдельный аккаунт — сейчас это текстовое поле в проекте
- SCORM-импорт курсов, адаптивное тестирование (Rasch), режим "locked browser"
- Kubernetes/микросервисы — сейчас управляемый Runable-хостинг (масштabируется автоматически)

## Технологический стек

Monorepo на Bun workspaces:

| Слой | Технология |
|---|---|
| Backend | Hono (TypeScript) поверх Bun, REST API |
| База данных | SQLite (Turso), Drizzle ORM |
| Auth | Better Auth (email+пароль, сессии/bearer-токены) |
| Web | React 19 + Vite + Tailwind CSS + Wouter + TanStack Query |
| Mobile | Expo + React Native + Expo Router (тот же API) |
| Desktop | Electron-обёртка (не задействована в этом MVP, доступна при необходимости) |

Веб и API работают в одном процессе (`packages/web`) — API отвечает на `/api/*`, фронтенд — на `/*`.
Мобильное приложение обращается к тому же API по HTTPS.

## Масштабируемость и надёжность (1000+ запросов)

- Bun — event-loop рантайм, эффективно держит тысячи одновременных соединений на одном процессе
- База данных — Turso (распределённый SQLite, реплики, низкая задержка), Drizzle ORM с подготовленными запросами
- Хостинг Runable автоматически масштабирует инстансы под нагрузкой
- Все тяжёлые операции (проверка теста, пересчёт прогресса) — короткие синхронные запросы без блокирующих циклов
- Для дальнейшего роста: вынести проверку тестов и подсчёт прогресса в очередь (например, через cron/queue), включить кэширование списков курсов/мероприятий (Redis или in-memory с TTL)

## Локальный запуск

Требуется Bun (https://bun.sh) и — для мобильного приложения — Expo Go на телефоне или эмулятор.

```bash
# из корня проекта
bun install

# веб (backend + frontend на одном порту)
bun run dev            # по умолчанию http://localhost:4200 (или другой свободный порт)

# мобильное приложение
bun run dev:mobile     # откроется Expo Dev Tools, отсканировать QR в Expo Go
```

### Переменные окружения

Лежат в `.env` в корне проекта (уже настроены для этого проекта в Runable):
`DATABASE_URL`, `DATABASE_AUTH_TOKEN` — Turso; `BETTER_AUTH_SECRET` — секрет сессий;
`WEBSITE_URL` — публичный адрес сайта. Мобильное приложение берёт адрес API из
`packages/mobile/app.json` → `expo.extra.apiUrl`.

### База данных

```bash
cd packages/web
bun run db:push        # применить схему к базе
bun run db:studio      # визуальный просмотр данных (Drizzle Studio)
bun run scripts/seed.ts   # наполнить демо-данными (курсы/тесты/мероприятия)
```

### Как выдать права администратора

По умолчанию все новые пользователи регистрируются как `student`. Чтобы сделать
пользователя админом:

1. Зайти в Drizzle Studio (`bun run db:studio`) → таблица `user` → найти пользователя → изменить `role` на `admin`.
2. Либо, если уже есть один админ — зайти в `/admin` на сайте и назначить роль другому пользователю через вкладку "Пользователи".

## Публикация мобильного приложения

Сборка `.apk`/`.aab` (Android) и `.ipa` (iOS) выполняется не в этой среде — на вкладке
предпросмотра мобильного приложения на платформе Runable есть кнопка **Publish**,
она подключается к вашему аккаунту Expo (EAS) и запускает сборку напрямую.

## Деплой веб-сайта

Публикация, домен и настройки хостинга веб-версии управляются через интерфейс платформы
Runable (кнопка Deploy/Publish в интерфейсе проекта) — отдельных действий в коде не требуется.
